Creating a new Document

################################################################################
# Create a new Royal Document
################################################################################
import-module royaldocument.powershell

# All work with Royal Documents are handled by a store which hold references to
# open documents
$RoyalStore = New-RoyalStore -UserName ($env:USERDOMAIN + '\' + $env:USERNAME)

# create the Royal Document
$RoyalDocumentPath = Join-Path -Path $env:USERPROFILE -ChildPath ('Documents\' + $env:USERDOMAIN + '.rtsz')
$RoyalDocument = New-RoyalDocument -Name $env:USERDOMAIN -FileName $RoyalDocumentPath -Store $RoyalStore
Note

The parameter "UserName" for the New-RoyalStore cmdlet is just for logging purposes and not for purposes like authentication.

Note

The RoyalStore acts a container for open documents. Features that reference other documents (e.g. Credentials referenced by Name or ID) are looking for all open documents in the store.

Specifying the document type

By default, the document has the type "Shared (Merge file)" which is a mode that supports multiple users working with the document.

You can check the current document type with the Get-RoyalDocumentType cmdlet.

Get-RoyalDocumentType -Document $RoyalDocument

You can change the mode with the Set-RoyalDocumentType cmdlet.

Set-RoyalDocumentType -Document $RoyalDocument -Type Merge
Set-RoyalDocumentType -Document $RoyalDocument -Type Overwrite

Saving the document

So far, this document exists only in memory. In order to store it on disc, you need to save it:

# save the Royal Document to disc and remove it from store
Out-RoyalDocument -Document $RoyalDocument
Close-RoyalDocument -Document $RoyalDocument
Note

The cmdlet Close-Document is removing the document from the Store.
